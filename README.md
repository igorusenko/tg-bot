# Telegram-бот записи (aiogram + SQLite)

Готовый бот для мастера по маникюру с:
- записью клиентов через inline-кнопки;
- админ-панелью;
- проверкой подписки на канал;
- автонапоминанием за 24 часа (APScheduler);
- хранением данных в SQLite.

## Установка и запуск

1. Создайте виртуальное окружение:
   - Windows:
     - `python -m venv .venv`
     - `.venv\Scripts\activate`
   - Linux/macOS:
     - `python3 -m venv .venv`
     - `source .venv/bin/activate`

2. Установите зависимости:
   - `pip install -r requirements.txt`

3. Создайте `.env` на основе `.env.example` и заполните:
   - `BOT_TOKEN` — токен бота;
   - `ADMIN_ID` — Telegram ID администраторов (список через запятую, например `123,456`);
   - `CHANNEL_ID` — ID канала (например, `-100...`);
   - `CHANNEL_LINK` — ссылка на канал;
   - `DB_PATH` — путь к SQLite БД (`bot.db` по умолчанию).

4. Запустите бота:
   - `python bot.py`

## Что реализовано

- Расписание на 1 месяц вперед.
- Запись пользователя: дата -> время -> имя -> телефон -> подтверждение.
- Ограничение: один пользователь может иметь только 1 активную запись.
- После записи слот блокируется; после отмены снова освобождается.
- Отправка уведомлений админу и в канал о новых/отмененных записях.
- Проверка подписки на канал перед записью:
  - кнопки `Подписаться` и `Проверить подписку`.
- Кнопка `Прайсы` (без FSM).
- Кнопка `Портфолио` со ссылкой на Pinterest.
- Админ-панель (доступ по `ADMIN_ID`, список админов):
  - добавление рабочих дней;
  - добавление/удаление слотов;
  - закрытие дня;
  - просмотр расписания на дату;
  - отмена записи клиента.
- Напоминание за 24 часа до записи через APScheduler:
  - создается автоматически при бронировании;
  - удаляется при отмене;
  - восстанавливается при перезапуске бота на старте.
